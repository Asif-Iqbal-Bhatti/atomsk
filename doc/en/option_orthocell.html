Option: orthogonal-cell

Syntax

-orthogonal-cell

Description

This option finds the optimal orthorhombic cell that corresponds to the system.

The initial system may have any arbitrary vectors H1, H2, H3. This option searches for linear combinations m⋅H1+n⋅H2+o⋅H3 (with m,n,o integers) that produce vectors aligned with the Cartesian directions X, Y, Z. Then, atom positions are duplicated using the translation vectors H1, H2, H3, so as to fill this new orthogonal cell. Finally, the initial cell vectors are replaced by the vectors of the orthorhombic cell. Since the new cell vectors are linear combinations of the initial cell vectors, they respect the lattice periodicity.

If shells (in the sense of an ionic core-shell model) and/or auxiliary properties are present, then they are also duplicated.

If a selection was defined (with the option -select), then the equivalent atoms in the new cell will be selected.

After this option is applied, the total number of atoms may differ from that of the initial system. In the particular case where box vectors are already all aligned with the Cartesian directions, but one or more of them point towards negative coordinates, this option will flip them so that they all point towards positive coordinates. The number of atoms will remain unchanged, but atoms will be translated by the appropriate box vector(s). Similarly, if H1 is aligned with the Cartesian Y axis and H2 with X, then this option will exchange these vectors so that H1 is aligned with X and H2 with Y.

It is recommended to call this option before duplicating the system with the option "-duplicate". Note that in certain conditions (for instance, if the initial box vectors form very small angles with Cartesian axes), then Atomsk may fail to find a suitable set of orthogonal vectors.

This option is parallelized with OpenMP. To control the number of parallel threads used by Atomsk, you may use the option -nthreads.

Default

By default, Atomsk preserves the cell vectors and does not try to find an equivalent orthogonal cell.

Examples

  • atomsk initial.cfg -rotate 45 z -orthocell final.cfg

    The initial system (initial.cfg) is rotated by 45Â° around the Z axis. Then, a suitable equivalent orthogonal cell is found, and atoms are duplicated to fill the new orthogonal cell. The final result will be output to final.cfg.
