Option: wrap

Syntax

-wrap

Description

This option wraps atoms back into the bounding box. In other words, all atoms that are not in the simulation box (defined by the simulation box vectors H) are translated by one or several box vectors until they end up inside of the simulation box.

If some atoms have shells (in the sense of an ionic core-shell model) then the shells will be wrapped independantly of cores, i.e. so that all cores and shells are inside of the box. So, if a core is inside of the box but its shell is not, then the core will remain at the same position but the shell will be wrapped and will be very far from the core. In order to fix that one can use the option -bind-shells.

If a selection was defined (with the option -select) then only selected atomswill be wrapped.

Default

By default atoms are not wrapped.

Examples

  • atomsk initial.cfg -wrap final.xyz

    This will read the file initial.cfg and wrap all atoms so that all coordinates are inside the simulation box. The result will be output into final.xyz.
