Option: expand

Syntax

-expand <ex> <ey> <ez>

-e <ex> <ey> <ez>

Description

This option allows to expand, or duplicate, a system in the three directions of space. It can be useful for (but not restricted to) building a supercell from a unit cell.

The translational vectors used to repeat the system are the base vectors of the original cell, noted Hx, Hy and Hz. The command-line parameters must be integers and, if they need any explanation at all:

  • ex: number of times the system is repeated along Hx;
  • ey: number of times the system is repeated along Hy;
  • ez: number of times the system is repeated along Hz.

If all values are equal to 1 then the system will remain the same. Any value equal to 0 is automatically replaced by 1, as a zero dimension would wipe out the whole system.

The replicas of the original system are appened to the original system. To change the order of atoms one can use the option -sort.

If original atoms possess auxiliary properties, the duplicate atoms will have the same auxiliary properties.

If a selection was defined (with the option -select) then only selected atoms will be duplicated.

Default

By default the system is not expanded.

Examples

  • atomsk unitcell.cfg -e 2 2 4 supercell.xyz

    This will create a 2x2x4 supercell from unitcell.cfg, and output it to supercell.xyz.

  • atomsk supercell.xyz unitcell.cfg -e 2 2 4

    Same as previous example. Note that the -e 2 2 4 statement cannot be broken, but intput and output file names can be specified before or after it.
