Option: dislocation

Syntax

-dislocation <p1> <p2> <disloctype> <dislocline> <dislocplane> <b1> [<b2> <b3>] <ν>

-disloc <p1> <p2> <disloctype> <dislocline> <dislocplane> <b1> [<b2> <b3>] <ν>

Description

This option allows to insert a straight edge or screw dislocation in the system. The equations used by this option can be found e.g. in J.P. Hirth and J. Lothe, Theory of dislocations.

The user has to provide the following parameters:

  • p1, p2: coordinates of the dislocation in the plane normal to the dislocation line (see dislocline), by order of permutation X, Y, Z (e.g. if dislocline=Y then p1 will be the position along Z and p2 the position along X).
  • disloctype: type of dislocation, must be "screw", "edge", or "edge2"; when using anisotropic elasticity, can be "mixed" (see below);
  • dislocline: direction of dislocation line, must be x, y or z;
  • dislocplane: direction normal to the plane of cut, must be x, y or z, and must be different from dislocline;
  • b1, b2, b3: components of the Burgers vector (in Å); for dislocations of type "edge", "edge2" and "screw" only one value must be given; for type "mixed" the 3 components must be provided;
  • ν: Poisson ratio of the material; it is very important to specify a correct value if you build an edge dislocation (for screw dislocations it is not used and can be set to any arbitrary value).

The positions <p1> and <p2> are usually given in Å. It is also possible to give them with respect to the box dimensions with the keyword BOX and an operation (see this page).

Edge dislocations can be constructed in two different ways that can be selected by setting the parameter disloctype to "edge" or "edge2". The first method, used when the parameter disloctype is set to edge, consists in inserting a row of new atoms in the system, and then applying atomic displacements corresponding to the elastic field of an edge dislocation. Subsequently the supercell vector in the direction of the Burgers vector is elongated by 2/3 of the Burgers vector -it is recommended to check it afterwards. Because this method changes the total number of atoms and the volume of the supercell, the comparison between the final and initial systems is made irrelevant. With the second method, used when the parameter disloctype is set to edge2, the edge dislocation is constructed by conserving the total number of atoms in the system (also conserving the supercell vectors). This method results in the presence of a step at one edge of the supercell, possibly messing up the boundary conditions.

For screw dislocations the total number of atoms as well as the cell vectors are unchanged.

By default the dislocations are introduced using the isotropic elastic solutions, i.e. the displacements applied to the atoms are, for an edge dislocation:

       ux1 = (b/2π) [ atan(x2/x1) + x1x2/(2(1-ν)(x12+x22)) ]
       ux2 = (-b/2π) [ (1-2ν)ln(x12+x22))/(4(1-ν)) + (x12-x22)/(4(1-ν)(x12+x22)) ]

and for a screw dislocation:

       ux3 = (b/2π) atan(x2/x1)

where x3 is the direction of <dislocline>, and x2 the normal to the plane of cut. The theoretical dislocation stresses (from the continuum theory) are also computed, however since the shear modulus μ is unknown the stresses are normalized to it, i.e. all stresses are calculated with μ=1 (in other words the quantity computed is actually σ/μ). The six Voigt components σxx, σyy, σzz, σxy, σyz and σxz are saved as auxiliary properties for each atom. If several dislocations are introduced in the system then the corresponding theoretical stresses are summed. Note that they can be written only to some files formats, like CFG (see this page for a list of formats that support writing of auxiliary properties).

The use of anisotropic elasticity is automatically triggered when the elastic tensor is defined before calling the present option, e.g. through the option -properties (see specifications of this option for details on the rotation of the elastic tensor). When using anisotropic elasticity a value of ν must still be given, but is not used (it can be set to zero or any arbitrary value). In addition to the dislocation types described above, a mixed dislocation can be built by setting disloctype to "mixed", after what the three components of the Burgers vector must be given. The equations of anisotropic elasticity are solved to determine the coefficients Ak(n), D(n), P(n) and Bijk(n), and the anisotropic displacements are applied:

       uk = ℜ{ (-2πi)-1 ∑(n=1,3) Ak(n) D(n) ln(x1+P(n)x2) }, k=1,3

These coefficients are also used to compute the theoretical dislocation stresses:

       σij = ℜ{ (-2πi)-1 ∑(n=1,3) Bijk(n) Ak(n) D(n) / (x1+P(n)x2) }

Since the elastic tensor is known, in this case it is the exact stresses that are computed. As for the isotropic case, the Voigt components are saved as auxiliary properties for each atom, and if several dislocations are constructed then their contributions to the stresses are added.

Beware that atomsk does not "automagically" find nor adjusts the Burgers vector of the dislocation, so a very precise value of b must be provided. Neither does the program find the optimal position for the dislocation center: a position (p1,p2) that exactly matches an atom position may result in unrealistic displacements, so you may have to play around with these coordinates to obtain proper results. As always, don't trust a program blindly -check your system before running any simulation, especially when building systems with dislocation(s).

If a selection was defined (with the option -select) then the displacements described above will be applied only to selected atoms.

Default

By default no dislocation is introduced at all.

Examples

  • atomsk initial.cfg -dislocation 0.5*BOX 0.5*BOX screw z y 3.2 0.0 final.xyz

    This will read initial.cfg and insert a screw dislocation of Burgers vector 3.2 Å lying along Z, in the middle of the box along X (because p1=0.5*BOX) and also in the middle along Y (because p2=0.5*BOX). The result will be output to final.xyz. Note that a value of the Poisson ratio must be provided although it is not used (it is arbitrarily set to 0.0 here)

  • atomsk initial.cfg -disloc 0.25*BOX 0.5*BOX screw z y 3.2 0.0 -disloc 0.75*BOX 0.5*BOX screw z y -3.2 0.0 final.xyz

    This will read initial.cfg and insert two screw dislocations of opposite Burgers vectors, the first one at (0.25;0.5) and the second one at (0.75;0.5). The result will be output to final.xyz.

  • atomsk unitcell.xyz -expand 30 2 20 -disloc 30 40.2 edge y x 2.8 0.28 dislocation.xsf

    This will read unitcell.xyz, expand it to form a 30x2x20 supercell, and then construct an edge dislocation by inserting a row of new atoms. The dislocation line will lie along Y, with a Burgers vector of 2.8 Å, using a Poisson ratio of 0.28, and the center of the dislocation will be placed at 30 Å along Z and 40.2 Å along X. The result will be output to dislocation.xsf.

  • atomsk unitcell.xyz -expand 30 2 20 -disloc 30 40.2 edge2 y x 2.8 0.28 dislocation.cfg

    Similar to the previous example, except that the edge dislocation will be constructued by adding a surface step (i.e. without inserting new atoms).

  • ctensor.txt

    # The elastic tensor for [100] [010] [001]
    elastic Voigt
    243.30 243.30 243.30
    145.00 145.00 145.00
    116.10 116.10 116.10

    # The crystal orientation
    orientation
    110
    1-10
    001

    atomsk initial.xyz -prop ctensor.txt -disloc 0.5*BOX 0.5*BOX screw y x 2.8 0.0 dislocation.cfg

    Assume we want to build a dislocation in a slab of iron with crystallographic orientation X=[110], Y=[110], Z=[001]. The original system (initial.xyz) has that orientation. In this example the option -properties is used to read the elastic tensor and the system orientation from the file ctensor.txt, therefore the anisotropic elasticity will be used to construct the dislocation. This is usually the most convenient way to use anisotropic elasticity for building a dislocation. Note that in the option -dislocation the Poisson ratio must be provided although it is not used (it is arbitrarily set to 0.0 here).

  • ctensor.txt

    # The elastic tensor for [100] [010] [001]
    elastic Voigt
    243.30 243.30 243.30
    145.00 145.00 145.00
    116.10 116.10 116.10

    atomsk initial.xyz -orient 110 1-10 001 100 010 001 -prop ctensor.txt -orient 100 010 001 110 1-10 001 -disloc 0.5*BOX 0.5*BOX screw y x 2.8 0.0 dislocation.cfg

    This example illustrates another way of doing the same thing as the previous example. The original system (initial.xyz which is oriented X=[110], Y=[110], Z=[001]) is first rotated thanks to the option -orient so that the [100] crystal direction lies along X, [010] along Y and [001] along Z. Then the elastic tensor, which corresponds to that orientation, is read. Then the system is rotated back to its original orientation (this could also be done by using the option -alignx instead), which also rotates the elastic tensor accordingly. Finally the dislocation can be introduced.

  • You may also want to look at the scripts in the "examples" folder provided with the program. The folder "Al_110dislocations" contains a bash script that builds two edge dislocations with opposite Burgers vectors in a slab of aluminium, using isotropic elasticity. The folder "Fe_disloc_screw111_anisotropy" shows how to build a 1/2<111> screw dislocation in iron using anisotropic elasticity.
