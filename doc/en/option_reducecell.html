Option: reduce-cell

Syntax

-reduce-cell

Description

This option attempts to reduce the system into a shorter cell.

This option parses the list of atoms to find pairs of equivalent atoms. If the vector joining two equivalent atoms is parallel to and shorter than an initial cell vector, then it replaces it.

This option preserves the crystal orientation and the cell vectors as they are, it just attempts to shorten them when it is possible. The final cell may not be the smallest unit cell, just the shortest along these directions.

Cell vectors may change after this option is applied. The positions and properties of atoms will not change, but if the cell is reduced then atoms that are outside of the final cell will be deleted, along with their shells and auxiliary properties if they had any.

If the system is rotated and you wish to obtain an orthorhombic unit cell, then use the option -orthogonal-cell before this one.

Note that this option is meant for use with perfect crystals. It cannot be expected to give good results in crystals containing defects, nor on non-crystalline or disordered systems.

Default

By default Atomsk does not reduce cell vectors.

Examples

  • atomsk system.cfg -reduce-cell xsf

    This will read the file system.cfg and re-calculate the supercell base vectors. The result will be output in system.xsf.

  • atomsk initial.cfg -rotate Z 45 -orthogonal-cell -reduce-cell final.cfg

    This will rotate the system by 45Â° around Z, search for an equivalent orthogonal cell, and finally attempt to reduce the size of this cell. The result will be output in final.cfg.
