Option: round-off

Syntax

-round-off <property> <threshold>

Description

This option allows to round off the values of the given <property> to the nearest integer. For the given <property>, if a value is closer than the <threshold> from the nearest integer, then the value is rounded off to the nearest integer. This option can be useful to eliminate round-off errors after constructing an atomic system, or after a simulation.

The <property> can be X, Y or Z (referring to Cartesian coordinates of atoms), or any per-atom auxiliary property. If it is defined as "XYZ", then all three coordinates will be rounded off; if it is "AUX", then all auxiliary properties will be rounded off. The <threshold> should be a positive real number.

Values that are different from the nearest integer by an amount larger than the given <threshold> will not be modified by this option.

If a selection was defined (with the option -select) then the rounding off will affect only the property of the selected atoms.

Default

By default numbers are not rounded off.

Examples

  • atomsk initial.cfg -round-off x 1e-6 final.cfg

    This will read the file initial.cfg, and round off the X coordinates of all atoms, if necessary. For instance, if an atom has a X coordinate equal to 0.00000023, it will be rounded off to zero. If an atom has a coordinate X=0.9999999967, it will be rounded off to 1, and so on. The final result will be saved into final.cfg.

  • atomsk initial.cfg -round-off xyz 1e-9 final.cfg

    This will read the file initial.cfg, and round off all coordinates (X, Y and Z) of all atoms, if they are closer to an integer than 10-9. The final result will be saved into final.cfg.

  • atomsk initial.cfg -round-off energy 5.0e-4 final.cfg

    This will read the file initial.cfg, and round off the auxiliary property called "energy", if it takes values closer to an integer than 5 10-4. The final result will be saved into final.cfg.

  • atomsk initial.cfg -round-off AUX 1.0e-8 final.cfg

    This will read the file initial.cfg, and round off all auxiliary properties. The final result will be saved into final.cfg.
