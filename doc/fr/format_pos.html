Format : POSCAR

Nom : positions atomiques pour Vienna Ab initio Simulation Package (VASP)

Extension : aucune (le fichier doit d'appeler POSCAR pour être utilisé avec VASP)

Spécification : documentation de VASP

Programmes de visualisation : Jmol, OVITO, VMD

Restrictions

Dans VASP, les espèces atomiques ne sont pas spécifiées dans le fichier POSCAR, mais dans le fichier de pseudopotentiels (POTCAR). Donc, lorsqu'il lit un fichier POSCAR, Atomsk vérifiera si un fichier POTCAR existe dans le dossier courant, et si oui l'utilisera pour définir les espèces atomiques. Sinon les atomes auront une espèce factice (le premier élément aura un numéro atomique égal à 1, le second à 2, etc.). Il est également possible d'utiliser l'option -substitute pour remplacer ces numéros factices par les numéros atomiques appropriés. Si le mot-clé "Selective dynamics" est utilisé, alors Atomsk enregistrera les coordonnées fixées des atomes comme propriétés auxiliaires.

De manière similaire, lors de l'écriture d'un fichier POSCAR les numéros atomiques seront perdus (il est dde la responsabilité de l'utilisateur d'utiliser u fichier POTCAR adéquat). De plus, dans le fichier POSCAR les atomes de même espèce doivent être contigus ; si ce n'est pas le cas Atomsk proposera de les ré-arranger (ce qui est identique à l'option -sort species pack). Si certains atomes sont fixes (parce que définis comme tels par le fichier d'entrée, ou par l'utilisation de l'option -fix) alors le mot-clé "Selective dynamics" sera utilisé, et les drapeaux correspondants seront ajoutés à chaque ligne de position atomique.

Exemples

  • atomsk unitcell.xsf -duplicate 1 1 4 -sort species pack POSCAR

    Ceci va lire le fichier unitcell.xsf, va le dupliquer pour former une super-cellule 1x1x4, puis triera les atomes selon leur espèce chimique, et écrira le résultat final dans le fichier VASP POSCAR.

  • atomsk CONTCAR -sub 1 Fe -sub 2 C final.xsf

    Ceci va lire le fichier VASP CONTCAR, substituera les atomes de type 1 par des atomes de fer et ceux de type 2 par du carbone, et écrira le résultat final dans final.xsf.
