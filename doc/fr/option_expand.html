Option : expand

Syntaxe

-expand <ex> <ey> <ez>

-e <ex> <ey> <ez>

Description

Cette option permet d'étendre, ou dupliquer, un système dans les trois directions de l'espace. Elle peut être utile par ex. pour construire une supercellule à partir d'une cellule élémentaire.

Les vecteurs de translation utilisés pour répéter le système sont ceux lus depuis le fichier d'entrée, notés Hx, Hy et Hz. Les paramètres associés avec cette option sont :

  • ex : nombre de répétitions du système suivant Hx;
  • ey : nombre de répétitions du système suivant Hy;
  • ez : nombre de répétitions du système suivant Hz.

Si toutes les valeurs sont égales à 1 alors le système reste identique. Toute valeur égale à zéro est automatiquement remplacée par 1, puisqu'une valeur de zéro suivant n'importe quelle dimension revient à ne plus avoir de système.

Dans le fichier de sortie, les répliques du système original sont ajoutées à la suite des positions originelles. Pour modifier l'ordre dans lequel les atomes apparaissent il est possible d'employer l'option -sort.

Si les atomes originels possèdent des propriétés auxiliaires, leurs répliques possèderont les mêmes valeurs de ces propriétés.

Si une sélection est définie (avec l'option -select) alors seuls les atomes sélectionnés seront dupliqués.

Défaut

Par défaut le système n'est pas étendu.

Examples

  • atomsk cellule_elementaire.cfg -e 2 2 4 supercellule.xyz

    Ceci lira le fichier cellule_elementaire.cfg, l'étendra pour former une supercellule de taille 2x2x4, et écrira le système final dans supercellule.xyz.

  • atomsk supercell.xyz unitcell.cfg -e 2 2 4

    Identique à l'exemple précédent. Notez que l'enchaînement -e 2 2 4 ne doit pas être cassé, en revanche les noms des fichiers d'entrée et de sortie peuvent apparaître dans n'importe quel ordre.
