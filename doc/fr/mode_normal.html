Mode : normal

Syntaxe

atomsk <fichier1> <fichier2> [options]

atomsk <fichier> <formats> [options]

atomsk <fichier> <préfixe> <formats> [options]

Description

Ce mode lit un fichier <fichier1> et le convertit vers le fichier <fichier2>. Si le <fichier2> a un nom ou une extension reconnaissable (par ex. .xyz ou .cfg) alors Atomsk écrira le <fichier2> dans le format correspondant.

Au lieu ou en plus de spécifier un nom de fichier de sortie, il est possible de spécifier un ou plusieurs <formats> de sortie, parmi ceux supportés par Atomsk. Le programme utilisera alors l'extension de ces formats avec le nom du fichier.

Si un <préfixe> (c'est-à-dire un nom de fichier sans extension) est spécifié alors il sera utilisé comme base pour les noms de fichiers de sortie.

Pour convertir tous les fichiers dans un dossier donné, il ne faut pas utiliser d'étoile (par ex. atomsk * cfg). À la place il est recommandé de créer une liste de noms de fichiers (par ex. avec ls * >liste.lst), et ensuite d'utiliser Atomsk en mode File list.

En plus de la conversion il est possible de transformer le système grâce aux options.

Exemples

  • atomsk monsysteme.gin exemple.cfg

    Ceci convertira le fichier monsysteme.gin vers exemple.cfg.

  • atomsk exemple.cfg monsysteme.gin

    Identique à l'exemple précédent, en supposant que le fichier monsysteme.gin existe et pas le fichier exemple.cfg. Le programme reconnaît quel fichier existe et l'utilise comme fichier d'entrée. Si les deux fichiers existent, le programme demande à l'utilisateur quel fichier doit être utilisé en entrée.

  • atomsk monsysteme.gin cfg

    Ceci convertira monsysteme.gin vers monsysteme.cfg.

  • atomsk monsysteme.gin nouveau_systeme cfg xsf

    Cet exemple utilise un préfixe (nouveau_systeme) et plusieurs formats de sortie. Le fichier monsysteme.gin sera converti vers nouveau_systeme.cfg et nouveau_systeme.xsf.

  • atomsk xsf nouveau_systeme cfg monsysteme.gin

    Identique à l'exemple précédent. Notez que les paramètres et noms de fichiers peuvent apparaître dans n'importe quel ordre dans la commande.

  • atomsk cellule.xyz -duplicate 2 2 6 -shift above 6.5 z 0.2 0.6 0.0 defaut_empilement.xsf

    Cet exemple utilise plusieurs options. Le fichier cellule.xyz sera lu, ensuite le système sera dupliqué pour former une super-cellule de dimensions 2x2x6, puis une partie du système sera translatée, et le système final sera écrit dans le fichier defaut_empilement.xsf.
