Option : dislocation

Syntaxe

-dislocation <p1> <p2> <typedisloc> <lignedisloc> <plandisloc> <b1> [<b2> <b3>] <ν>

-disloc <p1> <p2> <typedisloc> <lignedisloc> <plandisloc> <b1> [<b2> <b3>] <ν>

Description

Cette option permet d'insérer une dislocation rectiligne dans le système. Les équations employées par cette option se trouvent par ex. dans J.P. Hirth and J. Lothe, Theory of dislocations.

Les paramètres associés avec cette option sont :

  • p1, p2 : coordonnées de la dislocation dans le plan normal à la ligne de dislocation (voir lignedisloc), dans l'ordre de permutation X, Y, Z (par ex. si lignedisloc=Y alors p1 sera la position suivant Z et p2 celle suivant X).
  • typedisloc : type de dislocation, doit être "screw" pour une dislocation vis, "edge" ou "edge2" pour une dislocation coin ; lorsque l'élasticité anisotrope est utilisée, une dislocation mixte peut être construite en utilisant "mixed" (voir ci-dessous).
  • lignedisloc : direction de la ligne de dislocation, doit être x, y ou z.
  • plandisloc : direction normale au plan de coupure, doit être x, y ou z, et doit être différent de lignedisloc;
  • b1, b2, b3 : composantes du vecteur de Burgers (en Å) ; pour les dislocations de type coin ("edge" ou "edge2") ou vis ("screw") une seule valeur doit être donnée ; pour celles de type mixte ("mixed") les trois composantes doivent être données.
  • ν : coefficient de Poisson du matériau ; sa valeur est utilisée pour construire les dislocations coin ("edge" ou "edge2"), mais est ignorée dans le cas des dislocations vis ("screw") ou si l'élasticité anisotrope est utilisée.

Les positions <p1>, <p2> sont généralement données en Å. Il est également possible de spécifier une distance relative aux dimensions de la boîte avec le mot-clé BOX et une opération (voir cette page).

Les dislocations coin peuvent être construites de deux manières différentes, sélectionnables en donnant au paramètre typedisloc la valeur "edge" ou "edge2". La première méthode, employée lorsque typedisloc vaut edge, consiste à insérer un nouveau plan d'atomes dans le système, et à appliquer les déplacements atomiques correspondant au champ élastique d'une dislocation coin aux autres atomes. Ensuite la boîte est agrandie de 2/3 de la norme du vecteur de Burgers dans la direction du vecteur de Burgers -il est recommandé de vérifier la taille de boîte. Puisque cette méthode change le nombre total d'atomes et le volume de la boîte, la comparaison entre les systèmes initial et final devient impossible. La seconde méthode, employée lorsque disloctype vaut edge2, la dislocation coin est construite en conservant le nombre total d'atomes dans le système (et en conservant les vecteurs de boîte). Cette méthode résulte en une marche sur un côté de la boîte, ce qui peut modifier les conditions aux limites.

Lors de la construction d'une dislocation vis le nombre total d'atomes ainsi que les vecteurs de boîte sont conservés.

Par défaut les dislocations sont construites en utilisant les solutions élastiques isotropes, c-à-d. que les déplacements appliqués aux atomes sont, pour une dislocation coin :

       ux1 = (b/2π) [ atan(x2/x1) + x1x2/(2(1-ν)(x12+x22)) ]
       ux2 = (-b/2π) [ (1-2ν)ln(x12+x22))/(4(1-ν)) + (x12-x22)/(4(1-ν)(x12+x22)) ]

et pour une dislocation vis :

       ux3 = (b/2π) atan(x2/x1)

où x3 est la direction de <lignedisloc>, et x2 est la normale au plan de coupure. Les contraines théoriques (prédites par la théorie continue) sont également calculées, cependant puisque le module de cisaillement μ est inconnu ces contraintes sont normalisées, autrement dit la quantité calculée est en réalité σ/μ. Les six composantes de Voigt σxx, σyy, σzz, σxy, σyz et σxz sont enregistrées comme propriétés auxiliaires pour chaque atome. Si plusieurs dislocations sont introduites dans le système, ces contraintes sont cumulées. Notez que ces propriétés auxiliaires ne peuvent être écrites que vers certains formats de fichiers, comme le CFG (voir cette page pour une liste de formats supportant les propriétés auxiliaires).

L'élasticité anisotrope est automatiquement utilisée lorsque le tenseur élastique est défini avant que la présente option ne soit appelée, par ex. avec l'option -properties (se référer à cette option pour ce qui concerne la rotation du tenseur élastique). Lorsque l'élasticité anisotrope est utilisée une valeur du coefficient de Poisson ν doit toujours être fournie, mais elle n'est pas utilisée (elle peut donc être égale à zéro ou n'importe quelle valeur arbitraire). En plus des types de dislocations (coin et vis) décrits plus haut, une dislocation mixte peut être construite en donnant au paramètre typedisloc la valeur "mixed", après quoi les trois composantes du vecteur de Burgers doivent être fournies. Les équations de l'élasticité anisotrope sont résolues pour obtenir les valeurs des coefficients Ak(n), D(n), P(n) et Bijk(n), et les déplacements anisotropes sont appliqués :

       uk = ℜ{ (-2πi)-1 ∑(n=1,3) Ak(n) D(n) ln(x1+P(n)x2) }, k=1,3

Ces coefficients sont aussi utilisés pour calculer les contraintes théoriques associés à la dislocation :

       σij = ℜ{ (-2πi)-1 ∑(n=1,3) Bijk(n) Ak(n) D(n) / (x1+P(n)x2) }

Dans ce cas puisque le tenseur élastique est entièrement connu, ce sont bien les contraintes exactes qui sont calculées. Comme dans le cas isotrope, les composantes de Voigt sont enregistrées comme propriétés auxiliaires, et si plusieurs dislocations sont construites alors leurs contributions sont cumulées.

Attention : atomsk ne trouve ni n'ajuste "automagiquement" le vecteur de Burgers, aussi est-il impératif d'en fournir une valeur très précise. Le programme n'ajustera pas non plus la position de la dislocation vers un site particulier : si vous fournissez une position (p1,p2) qui correspond exactement à la position d'un atome, il peut en résulter des déplacements non réalistes. Il est possible que vous ayez à jouer sur ces positions pour obtenir des résultats cohérents. Comme toujours, ne croyez pas que le programme va penser pour vous -vérifiez votre système avant de lancer une simulation, surtout lors de la construction de dislocations.

Si une sélection est définie (avec l'option -select) alors les déplacements discutés ci-dessus ne seront appliqués qu'aux atomes sélectionnés.

Défaut

Par défaut aucune dislocation n'est introduite.

Exemples

  • atomsk initial.cfg -dislocation 0.5*BOX 0.5*BOX screw z y 3.2 0.0 final.xyz

    Ceci lira initial.cfg et insèrera une dislocation vis de vecteur de Burgers 3.2 Å suivant Z, au milieu de la boîte suivant X (puisque p1=0.5*BOX) et également au centre suivant Y (puisque p2=0.5*BOX). Le résultat sera écrit dans final.xyz. Notez que le coefficient de Poisson doit être fourni, bien qu'il ne soit pas utilisé pour une dislocation vis (il est arbitrairement égal à zéro ici).

  • atomsk initial.cfg -disloc 0.25*BOX 0.5*BOX screw z y 3.2 0.0 -disloc 0.75*BOX 0.5*BOX screw z y -3.2 0.0 final.xyz

    Ceci insèrera deux dislocations vis de vecteurs de Burgers opposés, la première en (0.25;0.5) et la seconde en (0.75;0.5) fois la taille de boîte.

  • atomsk unitcell.xyz -expand 30 2 20 -disloc 30 40.2 edge y x 2.8 0.28 dislocation.xsf

    Ceci lira unitcell.xyz, étendra le système pour former une supercellule de 30x2x20, puis construira une dislocation coin en insérant un nouveau plan d'atomes. La ligne de dislocation sera suivant Y, le vecteur de Burgers aura une amplitude de 2.8 Å, le coefficient de Poisson vaut 0.28, et le centre de la dislocation sera placé à Z=30 Å et X=40.2 Å. Le résultat sera écrit dans dislocation.xsf.

  • atomsk unitcell.xyz -expand 30 2 20 -disloc 30 40.2 edge2 y x 2.8 0.28 dislocation.cfg

    Similaire à l'exemple précédent, sauf que la dislocation coin sera introduite sans ajouter d'atome au système.

  • ctensor.txt

    # Le tenseur élastique pour l'orientation [100] [010] [001]
    elastic Voigt
    243.30 243.30 243.30
    145.00 145.00 145.00
    116.10 116.10 116.10

    # L'orientation du cristal
    orientation
    110
    1-10
    001

    atomsk initial.xyz -prop ctensor.txt -disloc 0.5*BOX 0.5*BOX screw y x 2.8 0.0 dislocation.cfg

    Supposons vouloir construire une dislocation dans un système de fer ayant pour orientation cristallographique X=[110], Y=[1-10], Z=[001]. Le système originel (initial.xyz) a bien cette orientation [110] [1-10] [001]. Dans cet exemple l'option -properties est utilisée pour lire le tenseur élastique et l'orientation du système depuis ctensor.txt, donc l'élasticité anisotrope sera employée pour construire la dislocation. Il s'agit souvent de la manière la plus simple d'utiliser l'élasticité anisotrope avec atomsk. Notez que dans l'option -dislocation le coefficient de Poisson doit être fourni bien qu'il ne soit pas utilisé ici (il est arbitrairement égal à zéro ici).

  • ctensor.txt

    # Le tenseur élastique pour l'orientation [100] [010] [001]
    elastic Voigt
    243.30 243.30 243.30
    145.00 145.00 145.00
    116.10 116.10 116.10

    atomsk initial.xyz -orient 110 1-10 001 100 010 001 -prop ctensor.txt -orient 100 010 001 110 1-10 001 -disloc 0.5*BOX 0.5*BOX screw y x 2.8 0.0 dislocation.cfg

    Cet exemple illustre une autre manière de faire la même chose que l'exemple précédent. Le système originel (initial.xyz ayant pour orientation [110] [1-10] [001]) est d'abord pivoté avec l'option -orient de sorte qu'il ait pour orientation X=[100], Y=[010] et Z=[001]. Puis le tenseur élastique, qui correspond à cette orientation, est lu grâce à l'option -properties. Puis le système est de nouveau pivoté pour lui redonner son orientation initiale (ceci aurait également pu être réalisé grâce à l'option -alignx), ce qui effectue également la rotation du tenseur élastique. Enfin la dislocation peut être introduite.

  • D'autres exemples sont fournis dans le dossier "examples" fourni avec le programme. Le dossier "Al_110dislocations" contient un script bash qui construit deux dislocations coin de vecteurs de Burgers opposés dans un système d'aluminium, en utilisant l'élasticité isotrope. Le dossier "Fe_disloc_screw111_anisotropy" montre comment construire une dislocation vis 1/2<111> dans du fer en utilisant l'élasticité anisotrope.
